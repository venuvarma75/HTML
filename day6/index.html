The latest stable release of Git is version 2.48.1, which was released on January 14, 2025.
Git is a distributed version control system (VCS) used for tracking changes in files and coordinating work among multiple people. It is widely used in software development but can be applied to any project that requires version tracking.
Three stages in git:-
1. Working Directory (Untracked/Modified State)
The Working Directory is the place where you actively work on files. It contains the actual files and directories that you can see and edit in your project folder.
Purpose: This stage is where you make changes to your project, such as creating, modifying, or deleting files.
State: Files in this stage are either untracked.
command: Git status.
2. Staging Area (Index)
The Staging Area is an intermediate space where you prepare changes before committing them to the repository.
Purpose: It acts as a buffer between your working directory and the repository, allowing you to review and selectively stage changes for a commit.
State: Files in the staging area are marked as ready to be committed. You can stage individual files or specific changes.
Commands:git add filename/git add .
3. Repository (Committed State)
The Repository is where all committed changes are stored in Git's database, forming the version history.
Purpose: This stage ensures that your changes are saved permanently, along with metadata such as the author, timestamp, and a commit message.
State: Files in the repository are part of the projectâ€™s official history and can be accessed or shared.
command:git commit -m"msg"/git log

GitHub is a web-based platform for version control and collaboration that uses Git as its underlying system. It allows developers to store, manage, and collaborate on code projects in a centralized location. GitHub is particularly popular for open-source and team-based software development.
Key Features of GitHub:
1.Version Control
2.repository

5.Issue Tracking and Project Management
6.Continuous Integration and Deployment (CI/CD)
7.Open Source Collaboration
8.Code Hosting and Sharing
9.Community and Documentation
10.Integration and Extensions
Commands:
1.git init:Initialize a new Git repository
2.git clone:clones the existing repository
3.git status:View the status of changes in the working directory
4.git add:Stage changes for commit.
5.git commit :Commit staged changes to the repository.
6.git push:Push local commits to a remote repository.
7.git branch:List or create branches.
8.git remote:Manage remote repositories.
9.git config:get Git configuration options.
10.sSgit pull:Fetch and merge changes from a remote repository.
